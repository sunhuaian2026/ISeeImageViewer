## 1. 一句话定位

> **跨文件夹的本地素材浏览面板 + 类似图查找**——Glance V1 "看图" 之上，叠一层"统一浏览多个本地文件夹 + 关键词搜索 + 以图搜图"。全本地、零上传、零订阅、不抢库（保留你磁盘原文件夹结构）。

### 差异化对位

| vs | 差异 |
|---|---|
| Apple Photos | 不强制 iCloud；不藏文件结构；不做 Memories/人脸识别 |
| Eagle | 不要求把素材 import 到自有库（**最大反对声音**：用户不想搬家）；零订阅 |
| Apple Preview | 提供智能文件夹 + 跨文件夹搜索 + 类似图查找 |
| Lightroom / Photo Mechanic | 不做 culling / 调色 / 编辑写回；不收费 |

### 主线 / 次线

- **主线 (c) 浏览**：日常打开 V2 → 看跨多个文件夹的"新积累"统一时间线
- **次线 (a) 找回**：偶尔需要找一张图 → 关键词搜索 + 类似图查找
- **不做线 (b) 整理**：culling 去重不在 V2 范围（决策 D10 / 不做清单）

## 2. 信息架构（IA）

### 2.1 Sidebar 布局

```
┌─ Sidebar ─────────────────────┐
│ ⚙️ 全部最近          [12]      │ ← 智能文件夹区（gear 图标，默认置顶，flat list，无 section header）
│ ⚙️ 本周新增          [3]       │   M1 ship 这两个；M3 加 "上个月" / "截图" / "大图"
│ ───────────────────────────── │ ← 细分隔线（视觉分组 cue，无文字标题）
│ ▼ 📁 工作素材                  │ ← V1 folder tree（hierarchical，folder 图标，保留 V1 行为）
│   📁 草稿                      │   右键菜单含 "在智能文件夹中隐藏" toggle
│   📁 输出                      │
│ ▶ 📁 截图                      │
│ ▶ 📁 表情包                    │
└────────────────────────────────┘
```

设计决策：

- **gear 图标 ⚙️** 区分智能文件夹与普通 folder（参照 macOS Finder / Notes convention）
- **默认置顶** 智能文件夹区位于 V1 folder tree 之上
- **无 section header 文字**，靠图标差异 + 细分隔线视觉分组
- **内置智能文件夹 fixed 顺序**（M1：全部最近 / 本周新增；M3：+ 上个月 / 截图 / 大图）；用户自定义在内置之下按创建时间倒序（M4）

### 2.2 启动默认状态

打开 V2 第一眼：自动选中「全部最近」→ 主 grid 显示所有受管文件夹的图，按 **birth time 倒序**，混合时间分段（**5 段**：今天 / 昨天 / 本周 / 本月 / 更早）。

### 2.3 主视图行为

| 用户在 sidebar 选中 | 主视图显示 |
|---|---|
| 智能文件夹（任一） | 该文件夹规则 query 结果 + 时间分段 grid |
| V1 folder tree 任一节点 | 单文件夹视图（V1 行为，**不变**） |
| 触发 ⌘F 搜索 | ephemeral 搜索结果视图（不进 sidebar），ESC 返回 |
| Quick Viewer 内点"找类似" | ephemeral 类似图结果视图，关闭返回 |

## 4. 数据模型（IndexStore schema）

Forward-looking schema：M1 一次性 ship M1-M3 已知字段，避免反复 ALTER TABLE。

### 4.1 主表 `images`

| 字段 | 类型 | M-stage | 说明 |
|---|---|---|---|
| `id` | Int PK | M1 | 主键 |
| `url_bookmark` | Blob | M1 | Security Scoped Bookmark |
| `birth_time` | Date | M1 | 排序主字段 |
| `file_size` | Int | M1 | 去重粗筛 + "大图" 规则 |
| `format` | String | M1 | 枚举（PNG/JPEG/HEIC/TIFF/WebP/RAW...） |
| `filename` | String | M1 | 搜索 + "截图" 规则 |
| `relative_path` | String | M1 | 相对受管 root 的 path |
| `folder_id` | Int FK | M1 | 关联 `folders` 表 |
| `dimensions_width` | Int | M1 | 顺手读，"大图" 规则用（M3） |
| `dimensions_height` | Int | M1 | 同上 |
| `content_sha256` | String? | M1 | lazy 仅在 size 碰撞时计算 |
| `dedup_canonical` | Bool? | M1 | 该字节内容的代表项 = true；副本 = false；唯一 = null |
| `feature_print` | Blob? | M2 | Vision 向量（~2-4KB） |
| `feature_print_revision` | Int? | M2 | Apple 算法版本号 |
| `supports_feature_print` | Bool | M2 | RAW / 矢量 false |
| `exif_capture_date` | Date? | M3 | 仅"按拍摄时间" smart folder 用 |

### 4.2 受管文件夹表 `folders`（**稀疏存储**）

| 字段 | 类型 | M-stage | 说明 |
|---|---|---|---|
| `id` | Int PK | M1 | 主键 |
| `root_url_bookmark` | Blob | M1 | Security Scoped Bookmark；仅 root folder 有值，subfolder 行 NULL |
| `relative_path` | String | M1 | 相对 root 的子路径；root 自身为空字符串 |
| `parent_root_id` | Int | M1 | 指向所属 root 的 `folders.id` |
| `hide_in_smart_view` | Bool | M1 | 该 folder 是否在智能文件夹中隐藏 |

**稀疏存储语义**：
- 每个 V1 root folder **必有一行**（root_url_bookmark 非 NULL，relative_path 空字符串）
- subfolder **仅在用户显式设过 hide 状态时有行**（绝大多数 subfolder 无行，节省空间）
- 一张图的 hide 计算：从该图所在 path 向上 walk，查 `folders` 表里有没有覆盖此 path 或祖先的行；找到第一条 = 该路径有效 hide 状态；没找到 = 默认 false
- 边界 case："hide root 后 unhide 特定子目录" → 写入 root 行 `hide=true` + 子目录行 `hide=false`，walk 时子目录行更具体优先

实现细节移到 writing-plans 阶段细化。

### 4.3 智能文件夹自定义规则（M4）

存 `~/Library/Application Support/Glance/smart-folders.json`，shape：

```json
{
  "version": 1,
  "smart_folders": [
    {
      "id": "uuid",
      "name": "我的设计参考",
      "rule": {
        "op": "AND",
        "children": [
          { "field": "format", "op": "IN", "value": ["PNG", "JPEG"] },
          { "field": "dimensions_width", "op": ">", "value": 2000 }
        ]
      },
      "created_at": "2026-..."
    }
  ]
}
```

JSON 格式 day-1 兼容嵌套（`{op, children}` 树形），未来 V3 GUI 支持嵌套时数据无需迁移。

### 4.4 Schema 版本与迁移

- IndexStore 启动检查 `schema_version` 字段
- 增量字段加：`ALTER TABLE ADD COLUMN`，老数据 NULL，后台 lazy backfill
- 迁移失败 → fallback 到 full re-index（drop + recreate + rebuild），用户看到一次性进度条
- Vision feature print revision 单独追踪：macOS 升级 revision 不匹配 → 仅 re-index `feature_print` 列，不影响其他字段

## 5. 智能文件夹规则引擎

### 5.1 表达力上限（D6）

支持 `字段 OP 值` 条件 + AND/OR 二层平铺组合。**不支持** 嵌套 / NOT。

### 5.2 内置 smart folder 规则定义

```ts
// "全部最近"（M1 默认）
{ op: AND, children: [
  { field: 'managed', op: '=', value: true },
  { field: 'hidden', op: '=', value: false },
  { field: 'dedup_canonical_or_null', op: '=', value: true }
] }
sort: birth_time DESC

// "本周新增"（M1）
{ op: AND, children: [
  { field: 'managed', op: '=', value: true },
  { field: 'hidden', op: '=', value: false },
  { field: 'dedup_canonical_or_null', op: '=', value: true },
  { field: 'birth_time', op: 'BETWEEN', value: ['-7d', 'now'] }
] }

// "截图"（M3）
{ op: AND, children: [
  ...common_filters,
  { op: OR, children: [
    { field: 'filename', op: 'CONTAINS', value: 'Screenshot' },
    { field: 'filename', op: 'CONTAINS', value: '截图' },
    { field: 'path', op: 'STARTS_WITH', value: '~/Desktop' }
  ] }
] }

// "大图"（M3）
{ op: AND, children: [
  ...common_filters,
  { op: OR, children: [
    { field: 'file_size', op: '>', value: 5_000_000 },  // 5MB
    { op: AND, children: [  // 注意：OR 内嵌 AND 是平铺二层结构，不算嵌套
      { field: 'dimensions_width', op: '>', value: 4000 },
      { field: 'dimensions_height', op: '>', value: 4000 }
    ] }
  ] }
] }

// "上个月"（M3）
{ op: AND, children: [
  ...common_filters,
  { field: 'birth_time', op: 'BETWEEN', value: ['last_month_start', 'last_month_end'] }
] }
```

### 5.3 时间分段算法（D4）

5 段固定：今天 / 昨天 / 本周 / 本月 / 更早

```
boundary_today    = Calendar.startOfDay(for: now)
boundary_yesterday = boundary_today - 1 day
boundary_thisweek = Calendar.firstDayOfWeek(containing: now)
boundary_thismonth = Calendar.firstDayOfMonth(containing: now)

bucket(image) =
  if image.birth_time >= boundary_today     → "今天"
  elif image.birth_time >= boundary_yesterday → "昨天"
  elif image.birth_time >= boundary_thisweek  → "本周"
  elif image.birth_time >= boundary_thismonth → "本月"
  else → "更早"
```

时区：device local，不做 UTC 归一化。

## 6. 找回（搜索 + 类似图）

### 6.1 关键词搜索（M3）

- 入口：⌘F（主窗口和 Quick Viewer 共用单一入口）
- Scope：全部受管文件夹（与智能文件夹一致）
- 普通用户：纯关键词输入 → filename / relative_path 全文 LIKE 匹配
- Power user：modifier 语法 `type:png size:>1mb` `birth:>2026-01-01` → parser 拆分后转 SmartFolder 规则结构
- 结果展示：`EphemeralResultView`，按 birth_time 倒序 + 时间分段
- 失效：搜索 0 结果 → "未找到匹配项 + 检查 spelling / 减少 modifier" 提示

### 6.2 类似图查找（M2）

- 入口：Quick Viewer toolbar "找类似" 按钮
- 算法：当前图 feature_print 与所有 `supports_feature_print=true` 的图算余弦距离，取 **top-30**（M2 写死，未来如有调整诉求改成 setting）
- 结果展示：`EphemeralResultView`（与搜索共用），按距离升序（最相似在前）
- 失效场景：
  - 当前图 `supports_feature_print=false`（RAW / 矢量 / 视频）→ 提示"该格式暂不支持类似图查找"+ 关闭按钮
  - 全库 feature print 索引未完成 → 提示"正在索引中，X% 完成，稍后再试"

### 6.3 "保存为智能文件夹"（M3+）

EphemeralResultView 顶部加按钮，点击后把当前搜索条件转成自定义 smart folder（M4 编辑器形态），写入 `smart-folders.json`，新条目出现在 sidebar 用户自定义段（内置之下）。**M1 / M2 不上**。

## 9. 验收标准（每 milestone）

每 milestone 完成时跑：

1. **三段式 verify**（`./scripts/verify.sh`）：编译 0 error 0 warning + 静态规则通过
2. **PENDING-USER-ACTIONS 人工测试清单**：本 milestone 新增的人工验证项（不能自动验证的视觉 / 性能 / 真实数据 case）追加到 `specs/PENDING-USER-ACTIONS.md`
3. **本 milestone 三标准核对**：(a) 端到端可跑（启动 V2 能用） (b) 用户可感知（截图能展示新能力） (c) 独立可 ship（minor 版本可发布）
4. **回归**：V1 既有功能不退化（QuickViewer / Inspector / 文件夹浏览 / 排序 / 键盘快捷键全部不变）
5. **性能目标**（非硬性 reject 标准，但应作为衡量基准）：1 万张图库下首次索引 < 10 分钟 / FSEvents 增量响应 < 5s / 智能文件夹 query < 200ms / Quick Viewer "找类似" < 1s（M2 后）。M1 mac 实测；超出基准要分析瓶颈再决定是否阻塞 ship

## 10. 关键架构决策索引

详细推理见 `specs/Roadmap.md`「关键架构决策」段 D1-D10。本节仅速查。

| ID | 决策 | 影响范围 |
|---|---|---|
| D1 | smart folder = rule-based ONLY，不存 manual membership | 整个 V2 哲学 |

## 2. M3 范围

### 2.1 做什么（M3 deliverables）

1. **Slice L（已 ship V2.2-beta1，2026-05-11）**— 3 个新内置 SmartFolder：上个月 / 截图 / 大图 + Calendar 自然月边界 token
2. **Slice M（V2.2 GA）** — 全局搜索
   - **SearchOverlayView** — 顶部 slide-in overlay，⌘F 触发；search field + 关闭按钮 + modifier hint 行
   - **SearchService** — input parser（Silent partial 容错）+ 编译成 `SmartFolderPredicate`
   - **⌘F 入口** — 主窗口（grid / preview / ephemeral）+ QuickViewer 两类层级
   - **3 个 modifier 语法**：`type:<format>` / `size:[>|<|=]<int>[b|k|m|g]` / `birth:[>|<|=]<ISO 日期>`
   - **结果展示** — 复用 `EphemeralResultView`（新加 `showTimeBuckets: Bool` toggle），按 birth_time 倒序 + 时间分段
   - **失效处理** — Silent partial 解析失败 token 当 keyword fallback；0 结果空态文案 "未找到匹配项 · 检查拼写或减少 modifier"

### 2.2 不做什么（M3 scope freeze）

- **"保存为智能文件夹"按钮** — v2-design § 6.3 明示 M3+，留给 M4 自定义 SmartFolder 编辑器
- **更多 modifier 字段**（path: / name: / width: / height:）— 三个已覆盖 v2-design § 6.1 全部举例，更多走 M4
- **modifier alias 表**（jpg vs jpeg / type:image shorthand）— 用户对 format 字符串严格 case-insensitive 比对
- **OR 多值 modifier**（`type:png,jpg`）— power user 多 type 走 keyword LIKE
- **FTS5 全文搜索** — 10k 库 SQL LIKE 实测 < 200ms 足够，FTS5 是性能优化未触红线不上
- **搜索历史 / saved searches / 自动补全** — 后期 power user feature，M3 不上
- **`include-hidden:` modifier** — 搜索严格继承 SmartFolder hide 规则（D18），用户对 hidden folder 改为显式 unhide
- **⌘F 在 QV 内不关 QV 直接叠 overlay** — 用户在 QV 按 ⌘F 同帧关 QV + 浮 overlay，避免 modal layer 叠加 race
- **input 内方向键 forward 到 grid** — input 内 ↑↓←→ 走 text cursor 导航，避免 modifier 输入时 conflict
- **click outside overlay dismiss** — overlay 不响应外部点击 dismiss，避免误触

### 4.3 焦点 / Modal layer 拓展

```swift
enum AppFocus: Hashable {
    case grid
    case preview
    case ephemeral
    case search   // M3 Slice M 新增

```

Modal layer 顺序：`QV > search > preview > ephemeral > baseGrid`。SearchOverlayView 自持 `.onKeyPress(.escape) { closeSearch() }`（mirror D15 子 view 自持 ESC 终态）。ContentView body 不加 `.search` 分支兜底（search 自处理）。

### 4.4 数据流（Search query lifecycle）

```
[user types in SearchOverlayView input]
   │
   ▼  searchInput @State 更新
   │
   ▼  SearchOverlayView .onChange(of: searchInput) → onInputChange(searchInput) callback
   │
   ▼  ContentView.onInputChange(input):
   │    searchTask?.cancel()                          ① cancel 上一轮
   │    searchTask = Task.detached(priority: .userInitiated) { [weak self] in
   │        try? await Task.sleep(for: .milliseconds(DS.Search.debounceMs))   ② 200ms debounce
   │        guard !Task.isCancelled else { return }
   │
   │        let parsed = SearchService.parse(input)
   │        let predicate = SearchService.compile(parsed)
   │        let folder = SmartFolder(predicate: predicate,
   │                                  sortBy: .birthTime, sortDescending: true,
   │                                  isBuiltIn: false)
   │        let compiled = try? SmartFolderQueryBuilder.compile(folder)
   │        let images = (try? store.fetch(compiled)) ?? []           ③ SQL LIKE
   │
   │        guard !Task.isCancelled else { return }
   │        let urls = computeUrls(images)                             ④ resolve bookmark
   │
   │        await MainActor.run { [weak self] in
   │            guard !Task.isCancelled else { return }
   │            self?.currentEphemeral = .search(query: input, results: urls)
   │        }
   │    }
   ▼
[UI 更新: EphemeralResultView 切到新 results，搜索 overlay 仍 visible]
```

**Cancel 三层保护**：
1. debounce sleep 前 `guard !Task.isCancelled`
2. SQL 完成 + bookmark resolve 后 `guard !Task.isCancelled`
3. MainActor.run 内最后 guard（async hop 后可能又 cancel）

**Enter 键路径**：跳过 200ms sleep 立即查询，方便 power user 快速确认。

## 5. SearchOverlayView 接口

```swift
struct SearchOverlayView: View {
    /// D15 单仲裁焦点 binding；overlay focus 拿走时 input 自动 active。
    @FocusState.Binding var focusTarget: AppFocus?
    /// 输入变化 callback：caller（ContentView）debounce + 触发 SearchService.parse → compile → fetch。
    let onInputChange: (String, _ skipDebounce: Bool) -> Void
    /// 关闭 callback：ESC / × button / 触发后 caller 清状态 + focusTarget = .grid。
    let onClose: () -> Void

    @State private var searchInput: String = ""

    var body: some View {
        VStack {
            HStack {
                Image(systemName: "magnifyingglass")
                TextField("搜索...", text: $searchInput)
                    .textFieldStyle(.plain)
                    .focused($focusTarget, equals: .search)
                    .onChange(of: searchInput) { _, new in onInputChange(new, false) }
                    .onKeyPress(.return) { onInputChange(searchInput, true); return .handled }
                    .onKeyPress(.escape) { onClose(); return .handled }
                Button(action: onClose) { Image(systemName: "xmark") }
            }
            Text("提示：type:png · size:>1mb · birth:>2026-01-01")
                .font(.caption)
                .foregroundStyle(.secondary)
        }
        .padding(DS.Search.overlayPadding)
        .frame(maxWidth: DS.Search.overlayMaxWidth)
        .background(.thickMaterial, in: RoundedRectangle(cornerRadius: DS.Search.overlayCornerRadius))
        .overlay(RoundedRectangle(cornerRadius: DS.Search.overlayCornerRadius).strokeBorder(.primary.opacity(0.12), lineWidth: 0.5))
        .padding(.top, DS.Spacing.md)
        .transition(.move(edge: .top).combined(with: .opacity))
    }
}
```

设计要求：
- input 内 `↑↓←→` 走 text cursor 导航（不 forward 到 grid，避免 modifier 输入 conflict）
- input 内 Enter = 跳 debounce 立即查
- ESC / × button = 同等 dismiss 路径，caller 清 `currentEphemeral` + `focusTarget = .grid`
- click outside overlay 不 dismiss（避免误触；overlay 不挂 `.onTapGesture { onClose() }` 在外层）
- modifier hint 行**始终可见**（教学性）

## 6. SearchService Parser 语法规范

### 6.1 三个 modifier 语法表

| Modifier | Syntax 示例 | Value 格式 | Operators 允许 | Maps to SmartFolderField |
|---|---|---|---|---|
| `type:` | `type:png`, `type:JPEG` | 单值字符串（case-insensitive），无逗号多值 | `=` 默认 | `.format` |
| `size:` | `size:>1mb`, `size:<500k`, `size:2gb` | 整数 + 可选单位 `b`/`k`/`m`/`g`（decimal 1000ⁿ，case-insensitive）| `>` / `<` / `=` 默认 | `.fileSize` |
| `birth:` | `birth:>2026-01-01`, `birth:<2026-05-01`, `birth:2026-04-15` | ISO 日期 `YYYY-MM-DD` 或带时区 `YYYY-MM-DDTHH:MM:SSZ` | `>` / `<` / `=` 默认 | `.birthTime` |

**Size 单位约定**（跟 macOS Finder 文件大小显示一致）：
- `b` = 1
- `k` / `kb` = 1,000
- `m` / `mb` = 1,000,000
- `g` / `gb` = 1,000,000,000

**Birth 单日 eq 语义**：`birth:2026-04-15` 翻译为 `betweenDuration [当日 00:00, 次日 00:00)` 复用既有 op。

### 6.2 Token → SmartFolderAtom 映射

```swift
// type:png →
.atom(.init(field: .format, op: .eq, value: .string("png")))

// size:>1mb (1mb = 1_000_000 B) →
.atom(.init(field: .fileSize, op: .greaterThan, value: .int(1_000_000)))

// birth:>2026-01-01 → 解析为 2026-01-01 00:00:00 UTC timestamp
.atom(.init(field: .birthTime, op: .greaterThan, value: .double(timestamp)))

// birth:2026-04-15 (单日 eq) → between [当日 00:00, 次日 00:00)
.atom(.init(field: .birthTime, op: .betweenDuration,
           value: .relativeTimeRange(start: "2026-04-15T00:00:00Z", end: "2026-04-16T00:00:00Z")))
```

### 6.3 Parser 算法（Silent partial）

```swift
struct ParsedSearch {
    var modifiers: [SmartFolderAtom]   // 成功解析的 modifier
    var keyword: String                // 剩余文本作 keyword (LIKE 模糊匹配)
    // 不暴露 parseErrors —— Silent partial 不报错（D17 决策）
}

func parse(_ input: String) -> ParsedSearch {
    let tokens = input.split(by: whitespace)
    var modifiers: [SmartFolderAtom] = []
    var keywordTokens: [String] = []

    for token in tokens {
        if let (field, expr) = parseFieldExpr(token) {   // r"^(\w+):(.+)$"
            switch field.lowercased() {
            case "type":
                if let atom = parseTypeValue(expr) { modifiers.append(atom) }
                else { keywordTokens.append(token) }     // 解析失败 fallback
            case "size":
                if let atom = parseSizeValue(expr) { modifiers.append(atom) }
                else { keywordTokens.append(token) }
            case "birth":
                if let atom = parseBirthValue(expr) { modifiers.append(atom) }
                else { keywordTokens.append(token) }
            default:
                keywordTokens.append(token)              // unknown modifier → keyword fallback
            }
        } else {
            keywordTokens.append(token)
        }
    }

    return ParsedSearch(modifiers: modifiers,
                        keyword: keywordTokens.joined(separator: " "))
}
```

### 6.4 编译成 SmartFolderPredicate

```swift
func compile(_ parsed: ParsedSearch) -> SmartFolderPredicate {
    var atoms: [SmartFolderPredicate] = [
        .atom(.init(field: .managed, op: .eq, value: .bool(true))),
        .atom(.init(field: .hidden, op: .eq, value: .bool(false))),          // D18 继承
        .atom(.init(field: .dedupCanonicalOrNull, op: .eq, value: .bool(true)))
    ]
    atoms.append(contentsOf: parsed.modifiers.map { .atom($0) })

    if !parsed.keyword.isEmpty {
        atoms.append(.or([
            .atom(.init(field: .filename, op: .contains, value: .string(parsed.keyword))),
            .atom(.init(field: .relativePath, op: .contains, value: .string(parsed.keyword)))
        ]))
    }

    return .and(atoms)
}
```

### 6.5 完整解析示例

| User input | Modifiers 抽出 | Keyword 剩余 | 最终 SQL |
|---|---|---|---|
| `screenshot` | — | `screenshot` | filename LIKE '%screenshot%' OR relative_path LIKE '%screenshot%' |
| `type:png` | `format = 'png'` | — | format = 'png' |
| `screenshot type:png` | `format = 'png'` | `screenshot` | format = 'png' AND (filename LIKE '%screenshot%' OR relative_path LIKE '%screenshot%') |
| `size:>1mb birth:>2026-04-01` | `fileSize > 1000000`, `birthTime > 2026-04-01 ts` | — | file_size > 1000000 AND birth_time > xxx |
| `screen type:invalidext` | — | `screen type:invalidext` | filename LIKE '%screen type:invalidext%' OR relative_path LIKE '%screen type:invalidext%' |
| `foo size:abc` | — | `foo size:abc` | filename LIKE '%foo size:abc%' OR relative_path LIKE '%foo size:abc%' |

最后两行：partial 失败 → 整个 token 拼回 keyword pool；用户看 0 结果或不相关结果，文案 hint 引导调整。

## 7. EphemeralResultView API 扩展（M3）

```swift
struct EphemeralResultView: View {
    let title: String                  // "类似于 xxx.png" / "搜索: xxx"
```

## 8. 性能 / 验收

### 8.1 性能预算（实测验收）

| 指标 | 目标 | 测法 |
|---|---|---|
| Search query 响应（10k 库典型 keyword）| < 200ms | 大库实测，记录 200ms / 500ms / 1s 分位 |
| SearchOverlayView open 动画 | < 16ms（一帧）| 视觉无感知抖动 |
| 200ms debounce 取消正确性 | rapid typing 无 stale 结果覆盖 | 手动验证 — 输入 "screen" 200ms 内连按多次 backspace + 重输不应残留旧结果 |
| Memory | search 期间无明显增长 | Activity Monitor 观察 < 5MB 浮动 |

如 query 响应超 500ms → 加 SQL index `(filename, relative_path)` 或切 FTS5 单 slice 优化（Slice N polish）。

## 11. 关键架构决策索引（M3 新增 D16-D20）

写入 `specs/Roadmap.md`「关键架构决策」段尾：

```markdown
### M3 决策（2026-05-11 brainstorming + plan-time 拍板）

29. **D16 M3 搜索 input = 顶部 Spotlight 式 overlay**：⌘F 触发顶部 slide-in overlay（detail 区顶端，max 600pt 居中），跟 EphemeralResultView 概念同源（D8 搜索 = ephemeral）；不挂 toolbar field、不弹 sheet modal。Why: (a) 跟 D8「搜索是 ephemeral」心智一致 — input / 结果 / 关闭 同一条退出路径；(b) toolbar field 在 QV 隐藏路径下失效需要 fallback overlay，复杂度叠加；(c) sheet 模态跟 D15 modal 层级再叠一层会重新触发焦点 race；(d) 用户在看图 app 内 ⌘F 心智更接近 Spotlight / 浏览器 ⌘F 而非 Mail / Finder。How to apply: SearchOverlayView 复用 D15 共享 `@FocusState focusTarget: AppFocus?` 加 `.search` case，layer 顺序 `QV > search > preview > ephemeral > baseGrid`，子 view 自持 ESC handler；后期加 history / saved searches 时再考虑升级 sheet 形态。

30. **D17 M3 搜索 modifier 解析 = Silent partial（不报错）**：`type:invalidext` / `size:abc` 等 modifier 解析失败时，把整个 token fallback 当 keyword LIKE 拼接，不在 UI 报错不阻挡查询。只解析 3 个 modifier：`type:` / `size:` / `birth:`。Why: (a) 实时查（200ms debounce）下错误提示会高频抖动 — 输 "type:p" 还没打完报错 → 烦人；(b) Silent fallback 容错最强，跟 Spotlight / GitHub / Slack search 同款行为；(c) 严格 spec 三个 modifier 已覆盖 v2-design § 6.1 全部举例 — power user 再要更多走 M4 自定义 SmartFolder。How to apply: SearchService.parse 返回 `(modifiers, keyword)` 永不抛错；解析失败的 token 自动进 keyword pool；UI 永不显示 parse error。如 type/size/birth 之外字段命名（如 `path:`）也按 unknown modifier 当 keyword fallback。

31. **D18 M3 搜索继承 SmartFolder hide 规则**：搜索 query predicate 强制带 `.hidden = false` atom（mirror 内置 SmartFolder 的 common_filters），hidden subfolder 中的图搜不到。Why: (a) D8 搜索 = ephemeral 智能视图，跟 SmartFolder 是同类聚合视图，hide 语义应一致；(b) 用户 hide 一个 folder 是显式表示"不想看到这里的图"，搜索"find anywhere"破坏该意图；(c) 实现零成本（复用既有 .hidden atom）。How to apply: SearchService.compile 出来的 SmartFolderPredicate 必带 `.hidden = false` + `.managed = true` + `.dedupCanonicalOrNull = true` 三个 common filter，跟 BuiltInSmartFolders.lastMonth / screenshots / largeImages 同款；future 如需 "include-hidden" 走 M4 自定义 SmartFolder 而非 modifier。

32. **D19 EphemeralResultView 加 `showTimeBuckets: Bool` toggle 解 spec 内部矛盾**：`v2-design § 6.1`（搜索按 birth_time 倒序 + 时间分段）vs `m2-design § 5`（找类似禁用时间分段，top-30 distance 排）的冲突由 EphemeralResultView 加可选时间分段参数解决。M3 search 传 true 走 sectioned LazyVGrid + pinned chip header（mirror SmartFolderGridView），M2 similar 维持 false 走 flat LazyVGrid。Why: (a) D8 已锁 EphemeralResultView 是搜索 + 类似图的统一组件，split 两个组件破坏 D8；(b) 时间分段对搜索（大结果集 + birth_time 排序）有意义，对类似图（top-30 distance 排序）无意义 — toggle 表达两种语义；(c) 同时加 `emptyStateText: String` 参数让 caller 控制空态文案（空 input / 0 结果 / 找类似无结果三种场景文案不同）。How to apply: EphemeralResultView API 加 `showTimeBuckets: Bool` 默认 false（保 M2 行为）+ `emptyStateText: String`；showTimeBuckets=true 时 internal 走 `groupedByTimeBucket()` helper + pinnedViews 渲染（已有 SmartFolderGridView 模式可借鉴）。

33. **D20 M3 SearchService = 新 `Glance/Search/` module（不扩 SmartFolderEngine）**：新增 `Glance/Search/` 目录含 `SearchService.swift` + `SearchInput.swift` + `SearchOverlayView.swift`；SearchService 只做 parser + 编译成 SmartFolderPredicate；最终 SQL 仍走 SmartFolderQueryBuilder + IndexStore.fetch 复用既有 query 路径。Why: (a) mirror M2 `Glance/Similarity/` module 边界 — search 是独立 feature 不污染 SmartFolderEngine；(b) SmartFolderEngine 当前职责仅 "已注册 SmartFolder 编译执行"，search 是 ephemeral 即时构造的 SmartFolder-shape 查询，语义不同；(c) 后期 M4 用户自定义 SmartFolder 编辑器复用 SearchService 的 parser（"保存搜索为 SmartFolder" → parser 输出 predicate 直接持久化）→ module 独立利于演化。How to apply: 新建 `Glance/Search/` 顶层，3 文件清单 fixed；SearchService.parse 输入 String → ParsedSearch struct（modifiers + keyword）；SearchService.compile(ParsedSearch) → SmartFolderPredicate；不需要新建 SearchQueryEngine（直接借 SmartFolderQueryBuilder + IndexStore.fetch）。
```

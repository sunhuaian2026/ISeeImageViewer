## 1. M2 一句话定位

> 在 V2.0 GA 跨文件夹浏览基础上叠一层「类似图查找」——Quick Viewer 内点「找类似」即在全库范围拉出视觉最相似的 top-30，纯 on-device Vision 推理，零外部费用、零上传。M2 的 demo 哇时刻。

## 2. M2 范围

### 2.1 做什么（M2 deliverables）

1. **SimilarityService** — Vision `VNFeaturePrintObservation` 包装；提取 feature print（Data + revision），用 Apple 自带 `computeDistance(_:to:)` 批量比较（指标为 Vision 内部实现，CONTEXT.md「俗称余弦距离」是行业速记说法，实际以 Apple API 返回 Float 为准；越小越相似）
2. **后台 feature print 索引** — 全库 `supports_feature_print=true` 的图后台逐张抽取 + 写 SQLite blob；进度 UI 复用 Slice I 进度 chip 样式
3. **Vision revision 迁移** — macOS 升级 `VNFeaturePrintObservation` revision 不匹配 → 仅 re-index `feature_print` 列，不污染其他字段
4. **Quick Viewer「找类似」按钮** — toolbar 新按钮，点击触发查询；unsupported 格式 disable
5. **EphemeralResultView 组件** — 临时结果视图骨架（M2 用于「找类似」，M3 复用给 ⌘F 搜索）；按距离升序，**top-30 写死**（不加阈值过滤，未来如需可改 setting）
6. **失效处理** — RAW/矢量/视频不支持 → 提示；feature print 全库未抽完 → 「已索引 X / Y 张，结果为部分库」banner

### 2.2 不做什么（M2 scope freeze）

- 自然语言搜索 / CLIP-shape embedding（D10 #4）
- OCR / 文本搜索（D10 #14）
- 像素级精确去重（已在 Slice H 完成）
- 「保存为智能文件夹」按钮（M3+ 才上）
- ⌘F 搜索（M3）
- 用户调 top-N（M2 写死 30）
- 距离阈值过滤（纯 top-30，不切「太不像就丢」）
- Schema 变更（M2 字段在 v1 schema 已 forward-looking 落地）

## 3. M2 plan-time 决策（D11-D14，写入 `specs/Roadmap.md` V2 决策段）

| ID | 决策 | 理由 |
|---|---|---|
| **D11** | 后台 feature print 索引触发时机 = **应用启动 + IndexStore 就绪后立即开抽**，已抽过的 lazy backfill | 用户首次点「找类似」时大概率已抽完；启动时 CPU 占用换"立刻可用"体验。给 K.4 性能验收留 backstop |
| **D12** | M2 拆 **2 slice**：Slice J（V2.1-beta1，后台索引 + 找类似端到端）+ Slice K（V2.1 GA，revision 迁移 + polish） | J 一次 ship 即可 demo；K 收 polish；保 vertical slice 三标准（端到端可跑 + 用户可感知 + 独立可 ship） |
| **D13** | top-30 = **纯 top-30 不加阈值过滤** | 实现简单 + 用户自己判断「像不像」；小库 30 后几张可能不相关，可后期按需加 setting 不影响 day-1 |
| **D14** | feature print 全库未抽完时 = **允许点击 + 仅查已抽取的 + 顶部 banner '已索引 X / Y 张'** | 部分可用体验，与 V2 spec 6.2 一致；避免 1 万图等 17 分钟 disable 死锁体验 |

## 4. 模块架构

### 4.1 新增模块

| 文件 | 职责 |
|---|---|
| `Glance/Similarity/SimilarityService.swift` | Vision `VNFeaturePrintObservation` 包装 + `computeDistance(_:to:)` 批量比较 + top-N。actor / nonisolated 边界 plan 阶段拍 |
| `Glance/Similarity/FeaturePrintIndexer.swift` | 后台索引 pipeline（mirror FolderScanner cancel/resume 模式）；启动 backfill + 订阅新增图入队；Task.isCancelled 检测；每 N 张 emit progress |
| `Glance/Similarity/FeaturePrintIndexingProgress.swift` | progress record（indexed / total / lastImageName） |
| `Glance/Similarity/FeaturePrintProgressView.swift` | chip 形态进度 UI，mirror Slice I `IndexingProgressView`（Capsule + thickMaterial + strokeBorder + X 取消） |
| `Glance/Similarity/EphemeralResultView.swift` | 临时结果视图骨架（layout + 复用 V1 ThumbnailCell + banner 槽 + 关闭按钮）；M2 用于「找类似」，M3 复用给搜索 |

### 4.2 扩展模块

| 模块 | 改动 |
|---|---|
| `IndexedImage`（IndexStore） | 加 5 method：`fetchImagesNeedingFeaturePrint(limit:)` / `setFeaturePrint(id, data, revision)` / `setFeaturePrintUnsupported(id)` / `fetchAllFeaturePrintsForCosine()` / `resetAllFeaturePrintsForRevisionMigration(targetRevision:)` |
| `IndexStoreHolder` | 加 `featurePrintProgress: FeaturePrintIndexingProgress?` @Published + `cancelFeaturePrintIndexing()` 钩子 + `featurePrintIndexer` 持有 |
| `QuickViewerOverlay` | toolbar 加「找类似」按钮；当前图 `supports_feature_print=false` → disable + tooltip "该格式暂不支持类似图查找" |
| `ContentView` | 加 `@State currentEphemeral: EphemeralRequest?`；非 nil 时 mainContent 换 EphemeralResultView 替代 baseGrid；ESC / 关闭按钮 / 切 sidebar entry → currentEphemeral = nil（QV 联动关闭） |
| `FolderStoreIndexBridge.handleCreated` | FSEvents 新增图后通知 FeaturePrintIndexer 入队 |
| `GlanceApp` | `.onAppear` → IndexStoreHolder isReady → FeaturePrintIndexer.start() |

### 4.3 数据流

**启动后台索引**：
```
GlanceApp.onAppear
  → IndexStoreHolder.isReady = true
  → FeaturePrintIndexer.start()
     loop:
       imgs = IndexStore.fetchImagesNeedingFeaturePrint(limit: 50)
       break if imgs.empty
       for img in imgs:
         if Task.isCancelled: break
         (data, rev) = SimilarityService.extract(url) ?? (nil, nil)
         if data == nil: setFeaturePrintUnsupported(img.id)
         else: setFeaturePrint(img.id, data, rev)
         emit progress every N images
```

**找类似**（用户路径）：
```
QV toolbar「找类似」点击
  → ContentView.handleFindSimilar(currentImageId)
  → SimilarityService.queryTopN(currentImageId, n: 30)
     - load currentImage feature_print → reconstruct VNFeaturePrintObservation
     - load all (id, feature_print) where supports_feature_print=true AND feature_print NOT NULL
     - for each candidate: reconstruct VNFeaturePrintObservation + computeDistance(_:to:)
     - sort ASC by distance + take 30 (excluding currentImageId)
  → fetch URLs by ids
  → ContentView.currentEphemeral = .similar(sourceUrl, results, banner)
  → EphemeralResultView render
  → ESC / 关闭 / 切 sidebar → currentEphemeral = nil
```

**FSEvents 新增图入队**：
```
FolderStoreIndexBridge.handleCreated(url)
  → IndexStore.insertImageIfAbsent(...)
  → IndexStoreHolder.featurePrintIndexer.enqueueIfNeeded(imageId)
```

## 5. EphemeralResultView 接口（M3 复用要求）

```swift
enum EphemeralRequest {
    case similar(sourceImageId: Int64, sourceUrl: URL, results: [URL], banner: String?)
    // M3: case search(query: String, results: [URL], banner: String?)
}

struct EphemeralResultView: View {
    let title: String          // "类似于 xxx.png" / M3 "搜索: xxx"
    let urls: [URL]            // 已排序
    let bannerText: String?    // "已索引 X / Y 张，结果为部分库"
    let onClose: () -> Void
    let onSelect: (URL) -> Void  // 单击预览 / 双击进 QV
    // 内部复用 ImageGridView ThumbnailCell pattern
}
```

设计要求：
- 不依赖 SmartFolder 概念（M3 搜索结果不是 SmartFolder）
- 不持久化（关闭即销毁状态）
- cell 行为 mirror SmartFolderGridView（单击预览 + 双击 QV + keyboard 支持），但禁用时间分段（top-30 是排序结果，不是时间序列）

## 6. Vision Revision 迁移逻辑（K.1）

启动检测：
```
currentRev = VNFeaturePrintObservation.requestRevision(currentMacOS)
storedRev = IndexStore.fetchSchemaMeta("feature_print_revision")  // schema_meta 表存
if storedRev != nil && storedRev != currentRev:
    log "Vision revision changed: \(storedRev) -> \(currentRev), re-indexing"
    IndexStore.resetAllFeaturePrintsForRevisionMigration(target: currentRev)
    // 该方法：UPDATE images SET feature_print=NULL, feature_print_revision=NULL
    //         WHERE supports_feature_print=1
    // indexer 启动后自动重抽（fetchImagesNeedingFeaturePrint 拉到这批）
IndexStore.setSchemaMeta("feature_print_revision", currentRev)
```

`schema_meta` 表用 K.1 task 内引入（key/value 单表）。仅影响 `feature_print` 列，不动其他字段。

## 7. 性能 / 验收

### 7.1 性能预算（K.4 实测验收）

| 项 | 预算 | 验证方式 |
|---|---|---|
| 单图 feature print 抽取 | < 200ms（M1 mac，HEIC/JPEG/PNG） | unit-style smoke test |
| 1 万图全库索引 | < 30 分钟（启动后后台跑） | 真实库实测，进 PENDING-USER-ACTIONS |
| 「找类似」查询响应 | < 1 秒（10k feature_print computeDistance + 排序） | 真实库实测 |
| feature_print blob 体积 | ~3-4KB / 张 × 10k = 30-40MB SQLite | `du` index.sqlite 验 |

非硬性 reject 标准，超出预算需分析瓶颈再决定阻塞 ship。

### 7.2 验收标准（M2 完成）

1. **三段式 verify**（`./scripts/verify.sh`）：编译 0 error 0 warning + 静态规则通过
2. **PENDING-USER-ACTIONS** 累积本 milestone 人工测试项（部分库 banner / 全库索引完成 / unsupported 格式提示 / revision 迁移 mock）
3. **三标准核对**：(a) 端到端可跑（启动 V2.1 → 后台索引跑 → QV 找类似） (b) 用户可感知（demo 哇时刻可截图） (c) 独立可 ship（V2.1 minor 可发布）
4. **回归**：V1 + V2 M1 既有功能不退化（QV / Inspector / smart folder grid / hide / FSEvents / dedup 全部不变）
5. **性能目标**：见 7.1

## 9. 已知风险 + 缓解

| Risk | 缓解 |
|---|---|
| 1 万图全库 feature print 抽取耗时 | 后台 Task.detached + 进度可见 + cancel + 跨 session resume（按 D11 启动时 lazy backfill） |
| feature_print blob 30-40MB SQLite | 直接存 blob 列；超 100k 图再考虑外置文件 |
| Vision 抽特征对 mmap Data 限制（HEIC/RAW/TIFF 表现不一） | 失败 → `supports_feature_print=false` 标记跳过；不阻塞 pipeline |
| EphemeralResultView 与 SmartFolderGridView 关系 | EphemeralResultView 仅 layout 容器复用 ThumbnailCell pattern；不引继承关系 |
| 「找类似」触发时全库未抽完 | 显式 banner "已索引 X / Y 张" + 不 disable 按钮（D14） |
| QV overlay 与 ephemeral 视图退场时序 | 复用 A.18 P0 transition .identity 模式，避免 baseGrid 闪现 |
| Vision revision 跨大版本变化（macOS 14 → 15 → 16） | K.1 启动比对 + 自动 reset re-index；不影响其他字段（D7 解耦保证） |
